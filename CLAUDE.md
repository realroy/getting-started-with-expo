## Commands

```bash
npm start          # Start dev server (interactive: choose iOS/Android/Web)
npm run ios        # Start in iOS simulator
npm run android    # Start in Android emulator
npm run web        # Start web version
npm run lint       # Run ESLint
npm run reset-project  # Move starter code to app-example/ and create blank app/
```

There is no test runner configured in this project.

## Architecture

This is an **Expo (React Native) app** using **Expo Router** for file-based routing, targeting iOS, Android, and Web from a single codebase.

### Routing

Routes live in `app/` and map directly to URL paths (like Next.js):
- `app/_layout.tsx` — root layout, sets up navigation stack
- `app/(tabs)/` — tab group; `_layout.tsx` configures the tab bar
- Screens inside `(tabs)/` become tabs automatically

### Platform-specific files

The project uses Expo's platform extension convention:
- `foo.ios.ts` — used only on iOS
- `foo.web.ts` — used only on Web
- `foo.ts` — fallback for Android and any unmatched platform

Examples: `hooks/use-color-scheme.ts` / `hooks/use-color-scheme.web.ts`, `components/ui/icon-symbol.tsx` / `components/ui/icon-symbol.ios.tsx`.

### Theming

- `constants/theme.ts` — defines color palettes for light and dark mode
- `hooks/use-theme-color.ts` — resolves a color name to the current theme's value
- `components/themed-text.tsx` / `themed-view.tsx` — drop-in replacements for `Text`/`View` that automatically apply theme colors

### Path aliases

`@/` maps to the project root. Use `import { X } from '@/components/...'` instead of relative paths.

### Experimental features enabled

- **New Architecture** (`newArchEnabled: true`)
- **React Compiler** (`reactCompiler: true`)
- **Typed Routes** (`typedRoutes: true`) — route strings are type-checked by TypeScript

### Native directories

`/ios/` and `/android/` are gitignored and generated by Expo on demand. Do not commit them.
